Documentation for SuperFlix

The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies. Specifically this app will focus on the superhero movie genre.

HTTP request details

1. General Home Page for App

  • Method type: GET
  • Query Perameters: '/'
  • Data in Body of Request: None
  • Response: Welome message for app

2. Return a list of ALL movies to the user

  • Method type: GET
  • Query Perameters: '/movies'
  • Data in Body of Request: None
  • Response: The JSON object containing all movie details

3. Return data (title, description, director, genre, URL, Featured Y/N) about a single movie by title to the user

  • Method type: GET
  • Query Perameters: '/movies/:title'
  • Data in Body of Request: None
  • Response: A JSON object holding the movie detials for the specific title. E.g.
    {title: "film name", description: "description", director: "director name", genre: "genre", imageURL: "URL", Featured: "Yes or No"}

4. Return data about a genre (description) by name/title (e.g., “Thriller”)

  • Method type: GET
  • Query Perameters: '/movies/genres/:genre'
  • Data in Body of Request: None
  • Response: All JSON objects relevant to specific genre selected in format E.g.
    {Name: "Genre Name", Description: "genre description"}

5. Return data about a director (bio, birth year, death year) by name

  • Method type: GET
  • Query Perameters: '/movies/directors/:director'
  • Data in Body of Request: None
  • Response: All JSON data relevant to specific director (including DOB, and other details if listed) selected in format E.g.
    {Name: "director name", Bio: "director bio", BirthYear: "yyyy", DeathYear: "yyyy", }

6. Allow new users to register

  • Method type: POST
  • Query Perameters: '/users/register'
  • Data in Body of Request: A JSON object holding the necessary data to create a new profile E.g.
    {name: "test name", email: "test@test.com", password: "test password", dateOfBirth: "test date"}
  • Response: A JSON object holding the name and email data for the new profile (not password or DOB). Now object will now include and display "id" as well E.g.
    {id: "new id", name: "test name", email: "test@test.com", password: "test password", dateOfBirth: "test date"}

7. Home page for each specific user

  • Method type: GET
  • Query Perameters: '/users/:id'
  • Data in Body of Request: None
  • Response: Welcome text message using user.name

8. Allow users to update their user info (username, password, email, date of birth)

  • Method type: PUT
  • Query Perameters: '/users/:id/updates'
  • Data in Body of Request: A JSON object holding the necessary data being updated in the profile E.g.
    {name: "new test name", email: "newtest@test.com", password: "newtest password", dateOfBirth: "new test date"}
  • Response: A text message specifying that the update was successful for specific field.

9. Allow users to add a movie to their list of favorites

  • Method type: POST
  • Query Perameters: '/users/:id/favorites/add/:title'
  • Data in Body of Request: None (movie object already exists in app)
  • Response: A text message confirming movie successfully added to list

10. Allow users to remove a movie from their list of favorites

  • Method type: DELETE
  • Query Perameters: '/users/:id/favorites/remove/:title'
  • Data in Body of Request: None
  • Response: A text messae confirming movie successfully deleted from list

11. Allow existing users to deregister

  • Method type: DELETE
  • Query Perameters: '/users/:id/deregister'
  • Data in Body of Request: None
  • Response: A text message confiming successful deletion of account.
